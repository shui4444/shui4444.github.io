来做一个小实验，代码如下：

public static void main(String[] args) throws InterruptedException {
  Thread thread = new Thread() {
    @Override
    public void run() {
      try {
        Thread.sleep(1000_1000);
      } catch (InterruptedException e) {
        e.printStackTrace();
      }
    }
  };
  thread.start();
  Thread.sleep(50_000);
}

执行以上代码，得出的结果就是main这个主线程结束了以后，创建的这个thread对象线程还在执行，这里通过jconsole来看

刚开始时：

主线程结束，创建的这个线程还在执行

对于这个程序，我希望在主线程结束时，这个名字为Thread-0的线程也跟着结束，这里就用到了本章介绍的daemon（守护线程）。创建的线程是守护线程的方式如下：

public final void setDaemon(boolean on)

源码的注释是这么描述的：

将该线程标记无论是守护线程或用户线程。 当只有线程上运行的Java虚拟机退出都是守护线程。
线程开始之前此方法必须被调用。

所以这里可以通过它来实现我想达到的预期，现在的代码：

public static void main(String[] args) throws InterruptedException {
    Thread thread = new Thread() {
      @Override
      public void run() {
        try {
          Thread.sleep(1000_1000);
        } catch (InterruptedException e) {
          e.printStackTrace();
        }
      }
    };
    thread.setDaemon(true);
    thread.start();
    Thread.sleep(50_000);
  }

对于一些场景中，一些不重要的操作就可以通过创建守护线程来实现，但是这里要注意一点：主线程可能已经执行完成，在它之中创建的线程如果是守护线程的话，可能还在Runnable状态而主线程结束，这个守护线程也跟着结束了

设置守护线程（setDaemon）的时候，一定要在start之前否则它会出现异常
