授权概念

权限系统通常要解决认证和授权

认证，即在应用中谁能证明他就是他本人。一般提供如他们的身份ID一些标识信息来表明他就是他本人，如提供身份证，用户名/密码来证明。
授权，也叫访问控制，即在应用中控制谁能访问哪些资源（如访问页面/编辑数据/页面操作等）。
前面我们分析的都是认证，即你是谁问题？

这篇我们来分析你能做什么的问题。

授权分析

权限是某个资源能否杯用户访问，而并不是能否显示的问题。

权限场景

通常公司的系统分对外系统 权限场景比较简单，而内部管理系统权限场景比较复杂（业务及需求经常变动）

权限控制

权限控制是基于用户-角色-权限这套体系，这个我在后面详细介绍。其实我们前面的浏览器项目已经有权限控制了我们再来看下

com.b4.mcr.auth.browser.BrowserSecurityConfig#configure

.antMatchers(
                       "/session/invalid",
                       browser.getRegistUrl(),
                       browser.getLoginUrl(),
                       browser.getSignUpUrl(),
                       SecurityConstants.DEFAULT_UNAUTHENTICATED_URL,
                       SecurityConstants.DEFAULT_VALIDATE_CODE_URL_PREFIX + "/*"
               )
               .permitAll()
               .antMatchers("/user/**").hasRole("ADMIN")

然后在demo模块中依赖与browser模块

<dependency>
           <groupId>com.b4</groupId>
           <artifactId>mcr-auth-browser</artifactId>
           <version>1.0</version>
       </dependency>

在登录页面进行登录之后访问：http://www.pinzhi365.com/user/me

页面会响应403，这里的意思是没有权限，这里是因为在之前写的McrUserDetailsService类中给用户的角色：

private SocialUser buildUser(String username) {
        String password = passwordEncoder.encode("123");
        return new SocialUser(username, password, AuthorityUtils.commaSeparatedStringToAuthorityList("admin,ROLE_USER"));
    }

这里我在BrowserSecurityConfig中要求访问/user/**URL的角色含ADMIN，所以页面403了，在这里修改一下：

private SocialUser buildUser(String username) {
     String password = passwordEncoder.encode("123");
     return new SocialUser(username, password, AuthorityUtils.commaSeparatedStringToAuthorityList("admin,ROLE_USER,ROLE_ADMIN"));
 }

这里注意，前面要加上ROLE_，然后再去访问http://www.pinzhi365.com/user/me这个URL是能正常访问了
