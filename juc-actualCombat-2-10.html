volatile的使用场景

状态量标记

volatile boolean start=true
while(start){
    //...
}

void close(){
    start=false;
}

如果这里不用volatile，其它线程调用了close()没有刷到主存，那这里的while循环就一直到自己的线程工作空间（线程高速缓存、线程工作内存）里面去读，读的一直是true一直在执行。

屏障前后的一致性

volatile init =true
----Thread 1----
obj = createObj()

----Thread 2----
while(!init){
    sleep()
}
useTheObj(obj)

这里的init=true肯定是发生在2个线程中间的，其它的不保证不重排序，但是init一定在obj之前
