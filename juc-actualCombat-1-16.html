上一章介绍了线程安全的问题，通过银行叫号的例子发现它在多线程情况下会对数据修改产生一些问题，其实在深层次可以把这个问题归结到线程的MemroyCache，它会在后续中介绍。上一章介绍的synchronized到底是什么？它会导致在多线程的时候在它里面的代码变成“单线程”，可以理解为：只要有一个线程进入了synchronized的区域，那其它线程在执行到这的时候需要等待，待上一个线程进入synchronized区域外才能进入，这就使得被synchronized的代码看上去和单线程没有区别了，所以在使用synchronized会影响效率，来看以下代码：

package com.shui.juc.chapter7;

import java.time.Duration;
import java.time.Instant;

/**
 * @author Shui
 * @date 2020/6/25 13:13
 */
public class SynchronizedTest {

  private static final Object LOCK = new Object();

  public static void main(String[] args) throws InterruptedException {
    Runnable runnable = () -> {
      synchronized (LOCK) {
        try {
          Thread.sleep(100_000);
        } catch (InterruptedException e) {
          e.printStackTrace();
        }
      }
    };

    Thread t1 = new Thread(runnable);
    Thread t2 = new Thread(runnable);
    Thread t3 = new Thread(runnable);
    Instant start = Instant.now();
    t1.start();
    t2.start();
    t3.start();
    t1.join();
    t2.join();
    t3.join();
    Instant end = Instant.now();
    System.out.println("耗时：" + Duration
        .between(start, end)
        .getSeconds() + "s");

  }

}

结果

耗时：300s
