策略模式案例

需求

计算工资税率

普通实现方式

package com.shui.juc.chapter2;

/**
 * 税收计算器
 *
 * @author Shui
 * @date 2020/6/17
 */
public class TaxCalculator {

  /**
   * 奖金
   */
  private final double bonus;
  /**
   * 薪水
   */
  private final double salary;

  public TaxCalculator(double bonus, double salary) {
    this.bonus = bonus;
    this.salary = salary;
  }

  public double calcuate() {
    return this.calcTax();
  }

  public double getBonus() {
    return bonus;
  }

  public double getSalary() {
    return salary;
  }

  protected double calcTax() {
    return 0.0d;
  }

}

测试

package com.shui.juc.chapter2;

/**
 * @author Shui
 * @date 2020/6/17
 */
public class TaxCalculatorMain {
  public static void main(String[] args) {
    TaxCalculator taxCalculator = new TaxCalculator(10000d, 200d) {
      @Override
      protected double calcTax() {
        return getSalary() * 0.1 + getBonus() * 0.15;
      }

    };
  }
}

假设需求改动：税率需要修改，如果程序特别复杂就会付出很多的代价，首先这里需要改变方法的内部，不符合开闭原则，这样做肯定不合理，下面通过策略模式实现

策略模式实现

定义一个计算策略接口

package com.shui.juc.chapter2;

/**
 * 计算策略接口
 *
 * @author Shui
 * @date 2020/6/17
 */
public interface ICalculationStrategy {

   double calculate(double salary, double bonus);

}

它的实现类：

package com.shui.juc.chapter2;

/**
 * @author Shui
 * @date 2020/6/17
 */
public class NewCalculationStrategy implements ICalculationStrategy {

  @Override
  public double calculate(double salary, double bonus) {
    return salary * 0.3 + bonus * 0.3;
  }

}
package com.shui.juc.chapter2;

/**
 * @author Shui
 * @date 2020/6/17
 */
public class SimpleCalculationStrategy implements ICalculationStrategy {

  @Override
  public double calculate(double salary, double bonus) {
    return salary * 0.1 + bonus * 0.15;
  }

}

税收计算器-修改版

package com.shui.juc.chapter2;

/**
 * 税收计算器
 *
 * @author Shui
 * @date 2020/6/17
 */
public class TaxCalculator2 {

  /**
   * 奖金
   */
  private final double bonus;
  /**
   * 薪水
   */
  private final double salary;
  private ICalculationStrategy calculationStrategy;

  public TaxCalculator2(double bonus, double salary, ICalculationStrategy calculationStrategy) {
    this.bonus = bonus;
    this.salary = salary;
    this.calculationStrategy = calculationStrategy;
  }

  public void setCalculationStrategy(ICalculationStrategy calculationStrategy) {
    this.calculationStrategy = calculationStrategy;
  }

  public double calcuate() {
    return this.calculationStrategy.calculate(salary, bonus);
  }

}

测试

package com.shui.juc.chapter2;

/**
 * @author Shui
 * @date 2020/6/17
 */
public class TaxCalculatorMain2 {

  public static void main(String[] args) {
    TaxCalculator2 taxCalculator2 = new TaxCalculator2(10000, 200, new SimpleCalculationStrategy());
    System.out.println(taxCalculator2.calcuate());
    taxCalculator2.setCalculationStrategy(new NewCalculationStrategy());
    System.out.println(taxCalculator2.calcuate());
  }

}

总结

实际上在Thread中也使用了这种方式，当你去调用start方法时，它去调用了Runnable接口，这和上面的策略模式是一样的做法，而通过继承Thread类重写run方法就是上面的普通实现方式
