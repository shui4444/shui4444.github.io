这一章介绍如何强制结束线程，举个例子：读一个网络资源、做一些比较耗时的事情，期望是30分钟执行完成，超过30分钟就认为是失败的就不等待它了，通过暴力的方式去结束，但是在JDK暴力的方式只有stop()，这个方式已经被弃用了，那么用什么方式来做这个呢？

package com.shui.juc.chapter6;

/**
 * @author Shui
 * @date 2020/6/25 10:18
 */
public class ThreadService {

  /**
   * 执行线程
   */
  private static Thread executeThread;
  private boolean finished = false;

  /**
   * 执行一个异步任务
   */
  public void execute(Runnable task) {
    executeThread = new Thread(() -> {
      Thread runner = new Thread(task);
      runner.setDaemon(true);
      runner.start();
      try {
        runner.join();
        finished = true;
      } catch (InterruptedException ignored) {
      }
    });
    executeThread.start();
  }

  /**
   * 关闭
   */
  public void shutdown(long mills) {
    long currentTime = System.currentTimeMillis();
    //如果没有执行结束
    while (!finished) {
      //超时
      if ((System.currentTimeMillis() - currentTime) >= mills) {
        System.out.println("任务超时，需要结束它");
        executeThread.interrupt();
        break;
      }
      try {
        executeThread.sleep(1);
      } catch (InterruptedException e) {
        System.out.println("执行线程被打断");
        break;
      }
    }
    finished = false;
  }

}

上面的代码通过守护线程的特性加上interrupt()通过join()感知被打断来实现，测试一下结果
