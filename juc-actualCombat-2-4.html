前言

之前有去介绍《单例模式》，分了7个例子去介绍，然后怎么去让类又是单例又能够实现懒加载并且性能比较好，然后使用了Holder、枚举的方式，然后介绍了wait set的概念，在每一个对象Object在一个Monitor时候都会有一个wait set，wait set它主要存放一些Thread，然后用其它的线程去唤醒它，然后出来，wait set是一个比较虚的概念，可能在内存中也并没有给它划分一个对列栈去存放东西。在这个章节去介绍一个volatile

package com.shui.juc.chapter1;

/** @author Shui */
public class VolatileTest {
  private static final int MAX_LIMIT = 5;
  private static volatile int INIT_VALUE = 0;

  public static void main(String[] args) {
    new Thread(
            () -> {
              int localVal = INIT_VALUE;
              while (localVal < MAX_LIMIT) {
                if (localVal != INIT_VALUE) {
                  System.out.printf(getName() + "值已更新为 [%d]\n", INIT_VALUE);
                  localVal = INIT_VALUE;
                }
              }
            },
            "reader")
        .start();
    new Thread(
            () -> {
              int localVal = INIT_VALUE;
              while (INIT_VALUE < MAX_LIMIT) {
                System.out.printf(getName() + "值已更新为 [%d]\n", ++localVal);
                INIT_VALUE = localVal;
                try {
                  Thread.sleep(500);
                } catch (InterruptedException e) {
                  e.printStackTrace();
                }
              }
            },
            "updater")
        .start();
  }

  private static String getName() {
    return Thread.currentThread().getName() + ":";
  }
}

结果

updater:值已更新为 [1]
reader:值已更新为 [1]
updater:值已更新为 [2]
reader:值已更新为 [2]
updater:值已更新为 [3]
reader:值已更新为 [3]
updater:值已更新为 [4]
reader:值已更新为 [4]
updater:值已更新为 [5]
reader:值已更新为 [5]

上面代码通过volatile2个线程之前能感知变化，如果去除volatile

private static int INIT_VALUE = 0;

结果

updater:值已更新为 [1]
updater:值已更新为 [2]
updater:值已更新为 [3]
updater:值已更新为 [4]
updater:值已更新为 [5]
...

这是什么原因？在reader这个线程看来INIT_VALUE它一直是0,而里面的localVal也是0，所以变成了死循环
